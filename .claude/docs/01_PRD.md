# 원천청년부 앱 — 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 앱 이름 | 원천청년부 |
| 플랫폼 | iOS, Android (Flutter) |
| 개발 단계 | Phase 1 (중보기도 MVP) |
| 목표 | 교회 청년부 내 파편화된 정보/기능을 단일 앱으로 통합 |

---

## 2. 배경 및 문제 정의

현재 원천교회 청년부는 다음 기능들이 각기 다른 채널에 분산되어 있음:

- 출결 관리 (별도 수기 또는 외부 시트)
- 카카오채널 발송 (채널 앱)
- 송리스트 확인 (별도 공유 채널)
- 중보기도 올리기/확인 (단체 채팅방 등)

**핵심 문제**: UX 파편화로 인해 청년부원이 여러 채널을 오가야 하며, 정보 접근성 및 참여도가 낮음.

---

## 3. 목표 사용자 (Target Users)

- 원천교회 청년부 등록 부원 (교적부 기준)
- 연령대: 대학생 ~ 청년
- 디지털 친화적, 스마트폰 주 사용자

---

## 5. 기능 요구사항

### 5.1 인증 (Authentication)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| AUTH-01 | 이름 + 비밀번호로 로그인 | P0 |
| AUTH-02 | 최초 로그인 시 기본 비밀번호(woncheon2025)로 로그인 가능 | P0 |
| AUTH-03 | 최초 로그인 성공 후 반드시 비밀번호 변경 화면으로 이동 | P0 |
| AUTH-04 | 변경된 비밀번호는 이후 로그인에 사용 | P0 |
| AUTH-05 | 교적부 미등록 이름은 로그인 불가 | P0 |
| AUTH-06 | JWT 기반 토큰 인증 (Access Token + Refresh Token) | P0 |
| AUTH-07 | 디바이스 토큰 등록 (푸시 알림용, iOS 권한 동의 후) | P1 |

**비밀번호 규칙**
- 변경 시 최소 8자 이상
- 기본 비밀번호(woncheon2025)와 동일한 값으로 변경 불가

### 5.2 중보기도

| ID | 요구사항 | 우선순위 |
|---|---|---|
| PRAYER-01 | 중보기도 작성 (제목 없음, 본문만) | P0 |
| PRAYER-02 | 작성 시 익명/실명 여부 선택 | P0 |
| PRAYER-03 | 익명 선택 시 목록/상세에서 작성자 "익명"으로 표시 | P0 |
| PRAYER-04 | 전체 중보기도 목록 조회 (최신순 페이지네이션) | P0 |
| PRAYER-05 | 중보기도 상세 조회 | P0 |
| PRAYER-06 | 본인이 작성한 중보기도 삭제 가능 | P1 |
| PRAYER-07 | 본인 외 타인의 글 삭제 불가 | P0 |
| PRAYER-08 | 본문 최대 글자 수: 500자 | P1 |

### 5.3 푸시 알림

| ID | 요구사항 | 우선순위 |
|---|---|---|
| NOTIF-01 | iOS 알림 권한 동의 요청 (최초 앱 실행 시) | P0 |
| NOTIF-02 | 권한 동의한 사용자에게만 푸시 발송 | P0 |
| NOTIF-03 | 주 1회 (토요일 저녁 또는 일요일 오전) 스케줄 발송 | P0 |
| NOTIF-04 | 발송 메시지: "이번 주 N개의 중보기도가 올라왔어요 🙏" | P0 |
| NOTIF-05 | 해당 주에 새 기도 0개이면 알림 미발송 | P1 |
| NOTIF-06 | 알림 탭 시 앱 실행 → 중보기도 목록으로 딥링크 | P1 |

## 6. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 응답 시간 | API 응답 p95 < 2s |
| 가용성 | AWS Serverless 기본 SLA 수준 |
| 보안 | 비밀번호 bcrypt 해싱, HTTPS only |
| 확장성 | DynamoDB Single Table 설계로 Phase 2/3 엔티티 추가 용이 |
| 오프라인 | 오프라인 대응 없음 (MVP 제외) |

- 댓글/반응 기능
- 중보기도 수정 기능
- 다국어 지원
