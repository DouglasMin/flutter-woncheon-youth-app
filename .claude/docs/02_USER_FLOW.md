# 원천청년부 앱 — 유저 플로우 (User Flow)

## 전체 플로우 개요

```
앱 실행
  └─ 스플래시 (로고 SVG 애니메이션)
       └─ 로컬 토큰 유효성 확인
            ├─ [유효] → 홈 화면
            └─ [없음/만료] → 로그인 화면
```

---

## 1. 앱 최초 실행 플로우 (신규 사용자)

```
앱 설치 후 첫 실행
  └─ 스플래시 화면 (약 2초, 로고 애니메이션)
       └─ 로컬 토큰 없음 확인
            └─ 로그인 화면
                 └─ [이름 + 기본 PW: woncheon2025 입력]
                      ├─ [교적부 미등록 이름] → 에러: "등록된 청년부원이 아닙니다"
                      ├─ [비밀번호 불일치] → 에러: "비밀번호를 확인해주세요"
                      └─ [인증 성공, isFirstLogin=true]
                           └─ 비밀번호 변경 화면 (필수, 스킵 불가)
                                ├─ [기본 PW와 동일] → 에러: "기존 비밀번호와 다르게 설정해주세요"
                                ├─ [8자 미만] → 에러: "8자 이상 입력해주세요"
                                └─ [변경 완료]
                                     └─ iOS 알림 권한 요청 다이얼로그
                                          ├─ [허용] → 디바이스 토큰 서버 등록 → 홈 화면
                                          └─ [거부] → 홈 화면 (알림 없이 계속 사용)
```

---

## 2. 재방문 사용자 플로우

```
앱 실행
  └─ 스플래시 화면
       └─ Access Token 유효성 확인
            ├─ [유효] → 홈 화면 (바로 이동)
            ├─ [만료] → Refresh Token으로 재발급 시도
            │    ├─ [재발급 성공] → 홈 화면
            │    └─ [재발급 실패/만료] → 로그인 화면
            └─ [없음] → 로그인 화면
                 └─ [이름 + 변경된 PW 입력]
                      └─ [인증 성공, isFirstLogin=false]
                           └─ 홈 화면 (비번 변경 없이 바로)
```

---

## 3. 홈 화면 플로우

```
홈 화면 (Bottom Navigation or 메뉴 그리드)
  ├─ 중보기도 [활성화]
  │    └─ → 중보기도 플로우 (섹션 4)
  ├─ 출결 [비활성화]
  │    └─ 탭 시 토스트: "추후 추가될 예정이에요 🙏"
  ├─ 송리스트 [비활성화]
  │    └─ 탭 시 토스트: "추후 추가될 예정이에요 🙏"
  └─ 기타 메뉴 [비활성화]
       └─ 탭 시 토스트: "추후 추가될 예정이에요 🙏"
```

---

## 4. 중보기도 플로우

### 4-A. 목록 조회

```
중보기도 탭 진입
  └─ 중보기도 목록 화면
       ├─ 게시물 목록 (최신순, 무한스크롤 or 페이지네이션)
       │    └─ 각 아이템: [작성자(익명/실명)] [작성 날짜] [본문 미리보기]
       ├─ 게시물 탭
       │    └─ 중보기도 상세 화면
       │         ├─ 본문 전체
       │         ├─ 작성자 (익명/실명)
       │         ├─ 작성일시
       │         └─ [본인 작성글] 삭제 버튼 노출
       │              └─ [삭제 확인 다이얼로그] → 삭제 → 목록으로 복귀
       └─ 우측 하단 FAB (작성 버튼)
            └─ → 중보기도 작성 화면 (4-B)
```

### 4-B. 작성

```
중보기도 작성 화면
  ├─ 본문 입력 (최대 500자, 현재 글자수 표시)
  ├─ 익명 여부 토글 (기본값: 실명)
  └─ 등록 버튼
       ├─ [본문 비어있음] → 에러: "내용을 입력해주세요"
       └─ [등록 성공] → 목록 화면으로 복귀 (새 글이 최상단에 노출)
```

## 5. 푸시 알림 플로우

```
[서버 - EventBridge 스케줄]
  └─ 매주 토요일 20:00 KST 트리거
       └─ Lambda: 이번 주 새 중보기도 수 집계
            ├─ [0개] → 알림 미발송, 종료
            └─ [1개 이상] → 디바이스 토큰 목록 조회
                 └─ SNS를 통해 FCM/APNs로 일괄 발송
                      └─ 메시지: "이번 주 N개의 중보기도가 올라왔어요 🙏"

[사용자 디바이스]
  └─ 알림 수신
       └─ 알림 탭
            └─ 앱 실행 → 중보기도 목록 화면으로 딥링크
```

## 6. 엣지 케이스 정의

| 케이스 | 처리 방식 |
|---|---|
| 앱 사용 중 Access Token 만료 | API 401 응답 시 자동으로 Refresh Token으로 재발급, 실패 시 로그인으로 리다이렉트 |
| 알림 권한 최초 거부 후 재요청 | 앱 내에서 재요청 불가 (iOS 정책). 설정 앱으로 안내하는 배너 노출 (선택) |
| 중보기도 목록 비어있을 때 | Empty state: "아직 올라온 기도가 없어요. 첫 번째로 나눠주세요 🙏" |
| 네트워크 오류 | 토스트: "네트워크 연결을 확인해주세요" |
| 본인이 아닌 글 삭제 시도 (API 레벨) | 403 응답, 클라이언트에서 삭제 버튼 자체를 미노출로 1차 방어 |
| 동일 이름 중복 교적부원 | 추후 운영 정책으로 해결 (Phase 1은 이름 단일 식별자, 중복 발생 시 수동 처리) |
